Python中的深拷贝与浅拷贝

发表于 2018-07-20 | 更新于: 2018-10-17 | 分类于 Python

Python默认复制的副本共享可能引发意想不到的结果。

默认做浅复制

  1. =直接赋值，传递的是对象的引用，相当于一个别名，其中一个改变另一个也会跟着变。

  2. 构造方法或[:]做的是浅复制（即复制了最外层容器），如果所有元素都是不可变的，那么这样没有问题，还能节省内存。但是，如果有可变的元素，可能就会导致意想不到的问题。

  3. copy模块提供的deepcopy和copy函数能为任意对象做深复制和浅复制。

  4. 深复制有时可能太深了。例如，对象可能会引用不该复制的外部资源或单例值。我们可以实现特殊方法 __copy__() 和 __deepcopy__()，控制 copy 和 deepcopy 的行为。

复制列表（或多数内置的可变集合）最简单的方式是使用内置的类型构造方法或[:]。例如：

140027413719624

可以看出，副本与源列表相等，但是二者指代不同的对象。

下面看看这段代码的效果：

l1: [3, [66, 44], (7, 8, 9), 100]
l2: [3, [66, 44], (7, 8, 9)]
l1: [3, [66, 44, 33, 22], (7, 8, 9), 100]
l2: [3, [66, 44, 33, 22], (7, 8, 9, 10, 11)]

➊ l2 是 l1 的浅复制副本。此时的状态如图。

➋ 把 100 追加到 l1 中，对 l2 没有影响。

➌ 把内部列表 l1[1] 中的 55 删除。这对 l2 有影响，因为 l2[1] 绑定的列表与 l1[1]是同一个。

➍ 对可变的对象来说，如 l2[1] 引用的列表， += 运算符就地修改列表。这次修改在l1[1] 中也有体现，因为它是 l2[1] 的别名。

➎ 对元组来说， += 运算符创建一个新元组，然后重新绑定给变量 l2[2]。现在， l1 和 l2 中最后位置上的元组不是同一个对象。

➊ 使用 copy 和 deepcopy，创建 3 个不同的 Bus 实例。
➋ 审查 passengers 属性后发现，bus1 和 bus2 共享同一个列表对象，因为 bus2 是bus1 的浅复制副本。

❹ bus3 是 bus1 的深复制副本，因此它的 passengers 属性指代另一个列表。
